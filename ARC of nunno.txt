Nunno Finance: Full System Architecture
Nunno Finance is not just an AI chatbot; it is a Proprietary Financial Intelligence Engine designed to educate beginners while providing professional-grade data analysis.

Core Philosophical Difference
While most AI apps are simple "GPT Wrappers" (they just send your question to ChatGPT), Nunno Finance uses a Service-Oriented Architecture (SOA). Before the AI even speaks, Nunno's "sub-brains" (services) gather raw data from the real world, synthesize it, and then hand it to the AI for final translation into beginner-friendly terms.

2. The Orchestration Layer (The "Nunno" Persona)
When a message comes in:

Intent Classification: Nunno determines which tools are needed (e.g., "User is asking about Bitcoin price -> need TA and Historical Events").
Parallel Execution: All required services run simultaneously to fetch fresh data.
Synthesis: The data is minified and fed into the AI LLM (OpenRouter/Claude).
Empathy translation: The LLM takes the raw data and explains it using analogies, simple terms, and "Beginner's Notes".
3. The Live Data Stream (WebSocket)
Nunno's frontend isn't static. It maintains a persistent WebSocket connection directly to the Binance exchange, providing 1:1 real-time price updates and live charts without the user needing to refresh.

4. Scalable Infrastructure
Database: SQLite for blazing-fast local queries of historical events.
Backend: FastAPI (Python) for high-performance streaming responses.
Frontend: Vite/React for a premium, responsive UI experience.
